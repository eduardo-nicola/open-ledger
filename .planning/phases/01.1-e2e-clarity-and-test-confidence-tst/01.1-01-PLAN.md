Criar arquivo novo `tests/e2e/README.md` com as seções abaixo e conteúdo preenchido (português):

1. `## Pré-requisitos` — bullets obrigatórios: Node/npm; `supabase start`; `supabase db reset` (mencionar seed para `test@open-ledger.local`); copiar `.env.example` → `.env.local` e preencher `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`; para modo password nada além disso; para Google listar `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` no Supabase local e variáveis `E2E_GOOGLE_*` opcionais.

2. `## Base URL` — uma linha explícita: os testes usam `http://127.0.0.1:3000` (igual a `playwright.config.ts`), não misturar com `localhost` para evitar cookies duplicados.

3. `## Caminho de confiança (Google OAuth real)` — declarar alinhado a D-01: login na UI com botão "Entrar com Google"; não apresentar `signInWithPassword` + cookies como equivalente de confiança.

4. `## Variação A — Google já logado` — passos numerados: (1) obter sessão Google no mesmo browser/perfil que o Playwright usará; (2) rodar setup com `E2E_AUTH_MODE=google`; (3) esperado: após clicar "Entrar com Google", chegar em `/profile` sem digitar email/senha de novo.

5. `## Variação B — Google não logado` — passos: `E2E_AUTH_MODE=google`; ausência de sessão Google; fluxo completo até `/profile`; se `E2E_GOOGLE_TEST_EMAIL` e `E2E_GOOGLE_TEST_PASSWORD` definidos, descrever que o setup tenta automação; senão, intervenção manual dentro do timeout `E2E_MANUAL_AUTH_TIMEOUT_MS` (default 15 min).

6. `## Modo password (apenas debug / velocidade)` — texto explícito que `E2E_AUTH_MODE=password` (ou omissão da env, pois o default no código é password) usa `signInWithPassword` + cookies via `@supabase/ssr` — **não** substitui validação do fluxo Google; usar para smoke rápido e CI sem segredos Google.

7. `## Matriz de execução` — tabela markdown com colunas: `Cenário`, `Comando`, `E2E_AUTH_MODE`, `Headless/headed`, `Notas`. Linhas mínimas:
   - Smoke local password: `npm run test:e2e:smoke`, `password` ou vazio, headless, "CI/local padrão"
   - Smoke Google: `npm run test:e2e:google`, `google` (script já exporta), headed Chrome, "requer segredos / sessão"
   - Subconjunto auth tags: `npm run test:e2e:auth`, conforme modo escolhido no env, headless salvo google

8. `## headed vs headless` — quando `playwright.config.ts` força headed (projeto setup com `E2E_AUTH_MODE=google`).

Incluir linha de rodapé: referência a `.planning/TESTING-INITIATIVE.md` para contexto da iniciativa.
